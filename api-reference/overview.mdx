The CloudThinker API provides programmatic access to all platform features, enabling automation, custom integrations, and advanced workflows.

## Base URL

```
https://api.cloudthinker.io/v1
```

## Authentication

All API requests require authentication using an API key:

```bash
curl -H "Authorization: Bearer <your_api_key>" \
  https://api.cloudthinker.io/v1/workspaces
```

### Getting an API Key

1. Navigate to **Profile > API Keys**
2. Click **Create API Key**
3. Name your key and set expiration
4. Copy the key (displayed only once)

<Warning>
Keep your API key secure. Never expose it in client-side code or version control.
</Warning>

---

## Rate Limits

API requests are rate-limited per API key:

| Plan | Requests/Minute | Requests/Day |
|------|-----------------|--------------|
| Starter | 60 | 1,000 |
| Professional | 300 | 10,000 |
| Enterprise | 1,000 | Unlimited |

Rate limit headers are included in all responses:

```
X-RateLimit-Limit: 300
X-RateLimit-Remaining: 299
X-RateLimit-Reset: 1704067200
```

---

## Response Format

All responses are JSON:

### Success Response

```json
{
  "data": {
    "id": "ws_abc123",
    "name": "Production",
    "created_at": "2024-01-15T10:00:00Z"
  }
}
```

### List Response

```json
{
  "data": [
    { "id": "ws_abc123", "name": "Production" },
    { "id": "ws_def456", "name": "Staging" }
  ],
  "pagination": {
    "page": 1,
    "per_page": 20,
    "total": 2,
    "total_pages": 1
  }
}
```

### Error Response

```json
{
  "error": {
    "code": "invalid_request",
    "message": "The workspace_id parameter is required",
    "details": {
      "field": "workspace_id"
    }
  }
}
```

## Pagination

List endpoints support pagination:

```bash
GET /workspaces/{id}/resources?page=2&per_page=50
```

### Parameters

| Parameter | Default | Max | Description |
|-----------|---------|-----|-------------|
| `page` | 1 | - | Page number |
| `per_page` | 20 | 100 | Items per page |

---

## Filtering

Most list endpoints support filtering:

```bash
GET /workspaces/{id}/recommendations?status=pending&effort=low
```

### Common Filters

| Filter | Description |
|--------|-------------|
| `status` | Filter by status |
| `created_after` | Items created after date |
| `created_before` | Items created before date |
| `sort` | Sort field |
| `order` | Sort order (asc/desc) |

## Webhooks via API

Create webhook subscriptions programmatically:

```bash
POST /workspaces/{id}/webhooks
Content-Type: application/json

{
  "name": "Jira Integration",
  "url": "https://your-system.com/webhook",
  "events": ["recommendation.created", "incident.created"],
  "secret": "your_webhook_secret"
}
```

## Example: Cost Analysis Automation

```python
import requests

API_KEY = "your_api_key"
BASE_URL = "https://api.cloudthinker.io/v1"

headers = {"Authorization": f"Bearer {API_KEY}"}

# Start a conversation with Alex for cost analysis
response = requests.post(
    f"{BASE_URL}/workspaces/ws_abc123/conversations",
    headers=headers,
    json={
        "agent": "alex",
        "message": "Analyze EC2 costs and generate recommendations"
    }
)

conversation = response.json()["data"]
print(f"Conversation started: {conversation['id']}")

# Get recommendations
response = requests.get(
    f"{BASE_URL}/workspaces/ws_abc123/recommendations",
    headers=headers,
    params={"status": "pending", "source": "conversation"}
)

recommendations = response.json()["data"]
for rec in recommendations:
    print(f"- {rec['title']}: ${rec['potential_savings']}/month")
```
