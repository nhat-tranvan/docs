Connect your Netlify account to enable CloudThinker agents to inventory sites across teams, check deploy status, and surface production URLs and project details.

Netlify authenticates with a single **personal access token (PAT)** scoped to a user's access.

---

## Prerequisites

- A **Netlify account** with access to the teams and sites you want to inventory.
- A **personal access token** created by a user with access to those teams and sites.

<Info>
A Netlify personal access token inherits the access of the user who creates it. Use a user whose team and site access matches what CloudThinker should see.
</Info>

---

## Setup

<Steps>
  <Step title="Open Netlify">
    Sign in to your Netlify account.
  </Step>
  <Step title="Create a Personal Access Token">
    Go to **User settings → Applications**, and under **Personal access tokens** select **New access token**:
    - **Description**: `cloudthinker`
    - **Expiration**: set a rotation window

    Copy the token immediately — it may not be shown again.
  </Step>
  <Step title="Add Connection in CloudThinker">
    Navigate to **Connections → Netlify** and enter:
    - **Personal Access Token**: the token you just created

    Click **Connect**. CloudThinker verifies the token and shows a **Connected** status.
  </Step>
</Steps>

<Warning>
Copy the personal access token immediately after creation. You'll need to create a new token if it's lost.
</Warning>

## Required Permissions

A Netlify personal access token carries the same access as the user who created it — it sees every team and site that user can reach.

<Tip>
Follow least privilege: create the token from a user whose team and site membership is limited to what CloudThinker needs to inventory.
</Tip>

---

## Agent Capabilities

Once connected, agents can:

| Capability | Description |
|------------|-------------|
| **Project Inventory** | List sites across the teams the token can access |
| **Team Visibility** | Group projects by team |
| **Deploy Status** | Report each site's latest deploy state |
| **Production URLs** | Surface each site's production URL |
| **Deploy Details** | Inspect a specific deploy by ID |

### Verify the Connection

```bash
@alex list my Netlify projects by team, show each production URL and latest deploy state, then #visualize the results
```

### Example Prompts

```bash
@alex list my Netlify sites grouped by team and #flag any with a failed latest deploy
@alex show the production URL and latest deploy state for each Netlify project
@alex inspect the latest deploy for my main Netlify site and summarize its status
```

## Troubleshooting

<Accordion title="Failed to get Netlify token">
The token is missing, invalid, expired, or unavailable. Create a new Netlify personal access token and reconnect.
</Accordion>

<Accordion title="401 / Unauthorized">
The token's user lacks access to the selected team or site. Use a token from a Netlify user with access to that team or site.
</Accordion>

<Accordion title="404 on a site or deploy ID">
The ID is wrong, or the token can't access that resource. Run the project inventory again and copy IDs from the discovery output.
</Accordion>

---

## Security Best Practices

- **Least-privilege user** - Create the token from a user limited to the teams and sites CloudThinker needs
- **Dedicated token** - Name the token for CloudThinker so it's easy to audit and revoke
- **Token expiration** - Set an expiration and rotate the token regularly
- **Revoke when unused** - Remove the token in Netlify if you stop using the connection
