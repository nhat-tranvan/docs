# Datadog

Connect your Datadog account to enable CloudThinker agents to search logs, query metrics, monitor infrastructure, and investigate incidents across your entire stack.

---

## Supported Platforms

| Platform | Support |
|----------|---------|
| **Datadog** | All plans (Free, Pro, Enterprise) |
| **Datadog US1** | `app.datadoghq.com` |
| **Datadog US3** | `us3.datadoghq.com` |
| **Datadog US5** | `us5.datadoghq.com` |
| **Datadog EU1** | `app.datadoghq.eu` |
| **Datadog AP1** | `ap1.datadoghq.com` |

<Warning>
GovCloud (US1-FED) is **not supported** by the Datadog MCP Server.
</Warning>

## Setup

CloudThinker connects to Datadog using OAuth for secure, scoped access.

<Steps>
  <Step title="Open CloudThinker">
    Navigate to **Connections → Datadog** in your CloudThinker workspace.
  </Step>
  <Step title="Start OAuth Flow">
    Click **Connect with Datadog**. You will be redirected to Datadog's authorization page.
  </Step>
  <Step title="Authorize CloudThinker">
    Sign in to your Datadog account and review the permissions CloudThinker is requesting. Click **Authorize** to grant access.
  </Step>
  <Step title="Connection Complete">
    You will be redirected back to CloudThinker. The connection status will show as **Connected**.
  </Step>
</Steps>

<Info>
CloudThinker uses OAuth to connect to Datadog — no API keys or application keys need to be manually configured.
</Info>

---

## Required Permissions

OAuth scopes are requested during authorization. CloudThinker requests:

### Minimum

- **Logs Read** — Search and analyze log data
- **Metrics Read** — Query infrastructure and custom metrics
- **Monitors Read** — View monitor configurations and statuses

### Recommended

All minimum permissions, plus:
- **Incidents Read** — Access incident data for investigation
- **Dashboards Read** — View dashboard configurations
- **Hosts Read** — List and inspect infrastructure hosts
- **APM Read** — Access trace data for performance debugging

## Agent Capabilities

Once connected, agents can:

| Capability | Description |
|------------|-------------|
| **Log Search** | Search and analyze logs across services and environments |
| **Metric Queries** | Query infrastructure, application, and custom metrics |
| **Incident Investigation** | Correlate logs, traces, and metrics for root cause analysis |
| **Monitor Status** | Check monitors in alert or warning states |
| **Infrastructure Overview** | List hosts and their health status |
| **APM Traces** | Analyze distributed traces for latency bottlenecks |

### Example Prompts

```bash
@alex search for error logs in the production environment from the last 15 minutes
@alex check for active incidents and investigate root causes
@alex analyze APM traces for the checkout service and identify latency bottlenecks
@alex list all hosts and check for any monitors in alert state
```

## Troubleshooting

<Accordion title="OAuth authorization failed">
- Ensure you have admin permissions in your Datadog organization
- Try clearing browser cookies and re-initiating the OAuth flow
- Check that third-party integrations are not disabled in your Datadog org settings
</Accordion>

<Accordion title="Missing data">
- Verify the authorized scopes include the data you are trying to access
- Check your Datadog retention settings — data may have been aged out
- Ensure the service or host is sending data to Datadog
</Accordion>

<Accordion title="Connection expired">
- OAuth tokens may expire — reconnect by navigating to **Connections → Datadog** and clicking **Reconnect**
</Accordion>

<Accordion title="Wrong Datadog site">
- CloudThinker auto-detects your Datadog site during OAuth
- If data appears missing, verify you authorized the correct Datadog organization
</Accordion>

## Security Best Practices

- **OAuth preferred** - CloudThinker uses OAuth, so no API keys are stored
- **Scoped access** - Only authorize the permissions CloudThinker needs
- **Regular review** - Periodically review authorized integrations in Datadog under **Organization Settings → OAuth Apps**
- **Revoke when unused** - Revoke CloudThinker's access if you no longer use the integration
