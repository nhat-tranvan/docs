# Atlassian

Connect your Atlassian products (Jira, Confluence) to enable CloudThinker agents to access project information, create issues, and reference documentation.

---

## Supported Products

| Product | Capabilities |
|---------|-------------|
| **Jira Cloud** | Read projects, issues, sprints; Create/update issues |
| **Jira Data Center** | Read projects, issues, sprints; Create/update issues |
| **Confluence Cloud** | Read spaces, pages; Search documentation |
| **Confluence Data Center** | Read spaces, pages; Search documentation |

---

## Setup

<Steps>
  <Step title="Generate API Token">
    Create an API token for authentication

    **For Atlassian Cloud:**
    1. Go to [Atlassian Account Settings](https://id.atlassian.com/manage-profile/security/api-tokens)
    2. Click **Create API token**
    3. Name: `CloudThinker`
    4. Copy the token immediately

    **For Data Center:**
    1. Go to **Profile → Personal Access Tokens**
    2. Click **Create token**
    3. Set appropriate expiration
    4. Copy the token
  </Step>
  <Step title="Identify Your Instance">
    Note your Atlassian instance URL:
    - Cloud: `https://your-domain.atlassian.net`
    - Data Center: `https://jira.your-company.com`
  </Step>
  <Step title="Add Connection in CloudThinker">
    Navigate to **Connections → Atlassian** and enter:
    - Instance URL
    - Email (Cloud) or Username (Data Center)
    - API Token
  </Step>
  <Step title="Select Products">
    Choose which products to connect:
    - Jira (for issue tracking)
    - Confluence (for documentation)
  </Step>
</Steps>

## Required Permissions

### Jira

| Permission | Purpose |
|------------|---------|
| **Browse Projects** | View projects and issues |
| **Create Issues** | Create new issues from recommendations |
| **Edit Issues** | Update issue status and fields |
| **Add Comments** | Add agent analysis as comments |

### Confluence

| Permission | Purpose |
|------------|---------|
| **View Spaces** | Browse space contents |
| **View Pages** | Read documentation |
| **Add Comments** | Add agent insights (optional) |

---

## Agent Capabilities

Once connected, agents can:

| Capability | Description |
|------------|-------------|
| **Reference Docs** | Search Confluence for procedures and policies |
| **Create Issues** | Generate Jira issues from findings |
| **Link Findings** | Associate recommendations with existing issues |
| **Track Progress** | Monitor issue status and updates |

### Example Prompts

```bash
@oliver #report security findings and create Jira issues for critical items
@alex create optimization tickets in PROJECT for cost savings >$1000/month
@anna search Confluence for incident response procedures
```

## Configuration Options

| Option | Description | Default |
|--------|-------------|---------|
| **Default Project** | Jira project for new issues | Required |
| **Issue Type** | Default issue type (Task, Bug, etc.) | Task |
| **Auto-create Issues** | Automatically create issues from findings | Disabled |

## Troubleshooting

<Accordion title="Authentication failed">
- Verify email/username is correct
- Check API token hasn't expired
- For Cloud: Use email, not username
- For Data Center: Use username, not email
- Ensure no spaces in token
</Accordion>

<Accordion title="Cannot access project">
- Verify user has Browse Projects permission
- Check project key is correct
- Ensure project isn't archived
- Confirm user is added to project
</Accordion>

<Accordion title="Cannot create issues">
- Verify user has Create Issues permission
- Check required fields are configured
- Ensure issue type exists in project
- Confirm project allows the issue type
</Accordion>

<Accordion title="Confluence search not working">
- Verify user has View Spaces permission
- Check Confluence is included in connection
- Ensure spaces aren't restricted
- Confirm CQL search is enabled
</Accordion>
